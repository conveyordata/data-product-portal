Where should schema + semantic models appear?

The current output tab has output ports (left) and technical assets (right). Where does the new content go?

A

Expand inline on the output port card

Clicking an output port card expands it in-place to show tables → columns and semantic models. Schema lives inside the existing card, no navigation required.

[▼ Finance Data Port]
  Tables: orders, invoices, customers
    orders: id (int), amount (decimal) …
  Semantic models: revenue_model (MetricsFlow)
B

Side panel / drawer

Clicking an output port opens a right-side drawer that shows the full schema and semantic models. The main two-column layout stays visible in the background.

[Output Ports] [Tech Assets] | [═══ Finance Data ═══]
                          | Tables / Semantic Models
C

Dedicated output port detail page

Clicking an output port navigates to its own page with tabs: Overview, Schema, Semantic Models. Full screen real estate for rich schema display.

/data-products/:id/output-ports/:portId
[Overview] [Schema] [Semantic Models]
  Table: orders → columns, descriptions, tags
D

New tab at data product level

A "Schema" tab is added alongside the existing Input Ports, Output Ports, Team, etc. tabs. Shows all output ports' schemas aggregated in one view.

[Overview] [Input Ports] [Output] [Schema ★] [Team]
Schema tab → grouped by output port → tables → columns
