Design: API Endpoints

Three endpoints under the existing output ports route

Base route: /v2/data_products/{data_product_id}/output_ports/{id}/cost

POST /cost — Push a cost record

Auth: Requires OUTPUT_PORT__UPDATE_COST action (new Casbin action, same pattern as OUTPUT_PORT__UPDATE_DATA_QUALITY)

Request body:

{
  "compute_cost": 45.00,
  "storage_cost": 30.00,
  "platform_overhead_cost": 15.00
}

Response: 201 with the created record (id + recorded_at + all costs)

GET /cost — Fetch cost history

Auth: No special auth (same as other GET endpoints on output ports)

Query params: limit (default 90, for ~3 months of daily pushes)

Response:

{
  "output_port_id": "...",
  "records": [
    {
      "id": "...",
      "recorded_at": "2025-04-01T00:00:00Z",
      "compute_cost": 45.00,
      "storage_cost": 30.00,
      "platform_overhead_cost": 15.00,
      "total_cost": 90.00
    },
    ...
  ]
}
GET /cost/upstream — Fetch upstream cost exposure

Scoped to the consuming data product — shows what this data product pays for its upstream dependencies

Response:

{
  "total_upstream_cost": 42.50,
  "upstream_output_ports": [
    {
      "output_port_id": "...",
      "output_port_name": "customer_events",
      "data_product_name": "CRM Platform",
      "full_cost": 170.00,
      "consumer_count": 4,
      "proportional_cost": 42.50
    }
  ]
}

Note: uses the latest cost record for each upstream output port. Output ports with no cost data are omitted.

Data product level — cost summary

GET /v2/data_products/{id}/cost — Total cost across output ports

Aggregates the latest cost record from each output port belonging to this data product.
