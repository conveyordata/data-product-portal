Design: Data Model

A new table stores cost records per output port — one record per push

New table: output_port_cost_records

SQLAlchemy Model — OutputPortCostRecord
Column Type Notes
idUUIDPK
output_port_idUUIDFK → datasets.id (CASCADE delete)
recorded_atDateTimeserver_default=utcnow(), indexed
compute_costNumeric(12,4)EUR
storage_costNumeric(12,4)EUR
platform_overhead_costNumeric(12,4)EUR

Total cost = compute + storage + platform_overhead (computed property, not stored)

Relationship to existing models

The Dataset model (output port) gets a new relationship:

cost_records: Mapped[list["OutputPortCostRecord"]] = relationship(
    cascade="all, delete-orphan",
    order_by="OutputPortCostRecord.recorded_at.desc()",
    lazy="raise",
)

Latest record is fetched in the service layer, not as a joined relationship (avoids loading full history on every output port load).

Upstream cost exposure — computed, not stored

No separate table. Computed on demand by the service layer:

  1. Find all approved InputPort records for the consuming data product
  2. For each upstream output port, fetch its latest cost record
  3. Divide by that output port's data_product_count (approved consumers)
  4. Sum across all upstream connections → proportional upstream exposure

Example: upstream output port costs €120/month with 4 consumers → this data product's share = €30/month
